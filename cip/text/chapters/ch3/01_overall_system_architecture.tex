\section{Overall System Architecture}
The overall architecture of the proposed Indian Sign Language recognition system is designed to process visual gesture input and convert it into meaningful textual and audio output. The system operates through a multi-stage deep learning pipeline that extracts both spatial and temporal information from the video sequence.

The complete workflow of the proposed system is illustrated in Fig.~\ref{fig:arch-proposed}, which presents the architecture diagram of the model and the interaction between different processing modules. As shown in Fig.~\ref{fig:arch-proposed}, the system begins with the acquisition of video input captured using a standard RGB camera. The input video is decomposed into individual frames through preprocessing. These frames are resized and normalized to maintain consistent spatial dimensions and improve numerical stability during model training and inference.

The processed frames are passed to a dual-stream feature extraction stage where the system learns complementary gesture evidence. In the implemented modular backend, the RGB stream uses a ResNet-based encoder (ResNet18 by default, with configurable ResNet34/50 variants), while the pose stream encodes aligned landmark vectors through an MLP-based pose encoder. In the checkpoint runtime path, inference can also operate in a pose-first configuration for low-latency deployment. Together, these streams capture appearance cues and geometric motion cues before fusion.

For mathematical consistency across the thesis, the RGB frame at time step $t$ is denoted by $X_t$ and the aligned pose-landmark vector is denoted by $P_t$. The corresponding extracted feature vectors are represented by $F_t^{\mathrm{rgb}}$ and $F_t^{\mathrm{pose}}$, respectively.

After feature extraction, the system applies gated multimodal fusion. The implemented fusion learns modality-specific gates for RGB and pose features and combines them into a unified spatiotemporal representation, followed by normalization for stable downstream temporal modeling.

The fused feature sequence is then processed by a sequence modeling component that captures contextual relationships between frames. The modular path uses a BiLSTM temporal model by default, while the deployed checkpoint path uses a causal temporal-convolution plus self-attention variant; Transformer-style encoding remains a configurable alternative. A Connectionist Temporal Classification (CTC) layer aligns predicted labels with input sequences without requiring precise frame-level annotations. Finally, the recognized gesture sequence is converted into readable text and optionally transformed into speech output using a text-to-speech module.

Thus, the architecture integrates multiple deep learning components to effectively capture gesture appearance, motion dynamics, and contextual relationships, enabling accurate real-time recognition of Indian Sign Language gestures. In Fig.~\ref{fig:arch-proposed}, green blocks represent established/core CSLR pipeline operations, blue blocks represent advanced modeling enhancements emphasized in this work, magenta blocks indicate project-specific integration and correction modules, and white blocks denote control and interface transitions between stages.

\subsection{Block Diagram}
\subsubsection{Video Input and Preprocessing}
The system begins with RGB video capture from a live camera or recorded input source and treats this stream as a time-indexed sequence rather than independent images. Because raw capture is noisy and device-dependent, preprocessing acts as a stabilization layer that enforces consistent quality before feature learning starts. In this stage, the stream is decomposed into frames, sampled at a controlled temporal rate, resized to model-compatible dimensions, and normalized to a stable intensity distribution. In addition, frame quality control and sequence standardization are performed so that each sample satisfies minimum temporal completeness. This design is essential for deployment because recognition quality is highly sensitive to early-stage timing drift, blur accumulation, and irregular sequence lengths. Therefore, preprocessing is not only a data cleaning step but also a reliability control stage that directly impacts downstream model confidence and inference latency.

\subsubsection{RGB Backbone Spatial Features}
The spatial stream uses an RGB backbone encoder to convert each normalized frame into compact high-information embeddings that preserve gesture-defining appearance cues. In the implemented backend, this role is handled by the \texttt{RGBStream} module with ImageNet-pretrained ResNet18 as the default backbone, while ResNet34/50 remain configurable options. The stream captures static and local visual attributes such as hand contour, palm orientation, finger spread, and signer-centric appearance context. This stage provides the primary appearance evidence for recognition and produces feature vectors that are dimension-aligned for multimodal fusion.

\subsubsection{Pose-Landmark Stream Features}
The second stream encodes pose-landmark tensors aligned with each RGB frame. In the modular path, landmark extraction and pose encoding produce geometric features that emphasize joint trajectories, hand articulation, and relative motion patterns. These pose features remain informative even when appearance cues degrade under illumination change, background clutter, or signer variability. Because RGB and pose streams remain time-synchronized, the system can perform frame-wise multimodal fusion with consistent temporal alignment.

\subsubsection{Multi-Feature Attention Mechanism}
The attention module performs adaptive feature prioritization across RGB and pose representations and acts as a selective filter for informative evidence. Instead of treating all frames and channels equally, the model learns relevance scores that elevate discriminative cues and suppress low-value activations caused by background clutter, minor occlusion, or capture instability. This mechanism is central to robust ISL recognition because informative segments are not uniformly distributed across a sequence. By concentrating model capacity on semantically dense regions, the attention stage improves signal-to-noise ratio before fusion and reduces error propagation to decoding layers. In practical terms, attention contributes to better confidence consistency and lower sensitivity to signer-specific style variations.

\subsubsection{Feature Fusion}
After weighting, the RGB and pose representations are fused into a single multimodal embedding that preserves complementary strengths from both streams. Fusion is designed to retain fine-grained appearance details from the RGB branch while simultaneously preserving geometric motion continuity from the pose branch. This unified representation is necessary for complex signs where either appearance-only or pose-only evidence is insufficient for reliable classification. The fusion stage also improves operational robustness by reducing dependence on a single modality under noisy conditions. In the proposed architecture, this block becomes the key transition point from low-level feature encoding to high-level sequence reasoning and is directly linked to recognition quality in continuous input settings.

\subsubsection{Sequence Modeling  }
The fused sequence is modeled with a temporal encoder that captures short- and long-range dependencies across time. The modular backend uses BiLSTM by default for stable bidirectional context aggregation, while the deployed iSign checkpoint path uses a causal temporal-convolution block with self-attention for low-latency online behavior. Transformer encoding remains an optional configuration for long-context experimentation. This stage converts fused multimodal features into context-aware embeddings that are aligned for CTC decoding.

\subsubsection{Temporal Alignment Using CTC}
Connectionist Temporal Classification (CTC) provides sequence alignment between input frame streams and output token streams without requiring strict frame-wise annotations. This is critical for ISL datasets where exact gesture boundary labeling is expensive and often inconsistent across annotators. CTC allows the model to learn many-to-one temporal mapping with blank handling and repetition collapse, making it naturally suited for continuous recognition. By removing the need for exact frame-level supervision, CTC improves scalability and simplifies training data preparation. In the overall design, this alignment layer bridges context-rich sequence embeddings and stable gesture token output, thereby enabling practical training and inference under real-world temporal variability.

\subsubsection{Decoding Strategy (Beam and Greedy)}
During inference, decoding strategy is selected according to operating mode. Full-video inference uses beam search to evaluate competing token paths and improve sequence stability, while frame-sequence and live-stream modes typically use greedy decoding for lower latency. This mixed strategy balances accuracy and responsiveness under real-time constraints. Beam width and confidence thresholds are adjustable so the decoder can be tuned for deployment priorities before language-level correction.

\subsubsection{Language and Grammar Refinement}
The recognized gesture token sequence is mapped into text and then passed through a grammar refinement stage to improve readability, syntactic quality, and communication clarity. Since gloss ordering often differs from standard written language structure, direct token-to-text mapping may produce incomplete or structurally awkward output. The refinement stage performs normalization of word order, removal of repetitive artifacts, and sentence-level correction to produce user-facing text that is easier for non-signers to understand. This stage is essential for practical assistive use because recognition quality alone does not guarantee communicative quality. In effect, language refinement transforms technical recognition output into socially usable content.

\subsubsection{Output and Text-to-Speech}
The final stage renders refined text to the interface and, when enabled, synthesizes speech through a text-to-speech subsystem. Caption rendering supports immediate visual feedback and interaction continuity, while audio output enables communication with listeners who depend on spoken-language output. This dual-mode design improves accessibility and broadens practical usability in educational, service, and public-facing scenarios. The output subsystem also acts as an endpoint for confidence-aware validation, ensuring that unstable predictions can be corrected before presentation. Consequently, the architecture closes the loop from visual capture to multimodal human-consumable communication output.
