\section{Test Cases}
This section defines a practical test-case framework for validating the implemented backend at functional, interface, and robustness levels. The goal is to verify that the system produces correct outputs for valid requests, returns stable error codes for invalid inputs, and maintains usable performance for real-time operation.

\subsection{Test Scope and Strategy}
Testing is organized into three levels so that failures can be localized quickly:
\begin{itemize}[leftmargin=1.5em]
\item \textbf{Service health tests:} verify backend availability and runtime readiness.
\item \textbf{Inference API tests:} validate request/response behavior for video and frame endpoints.
\item \textbf{Pipeline behavior tests:} verify decoding schema, confidence range, and streaming response continuity.
\end{itemize}

In this project, the primary externally testable interfaces are:
\begin{itemize}[leftmargin=1.5em]
\item \texttt{GET /api/v1/health}
\item \texttt{GET /api/v1/health/system}
\item \texttt{GET /api/v1/health/models}
\item \texttt{POST /api/v1/inference/video}
\item \texttt{POST /api/v1/inference/frames}
\item \texttt{WS /api/v1/ws/inference}
\end{itemize}
